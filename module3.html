HTML Lists: Ordered and Unordered Lists

  1. Unordered Lists (<ul>)

    2. An Unordered List is used when the order of items doesn't matter like a grocery list or to-do list. These lists display items with bullet points.

    Basic Structure:

      <ul>
        <li>Milk<li>
        <li>Bread<li>
        <li>Eggs<li>
      <ul>

    • <ul>: Starts the Unordered list (stands for unordered list)
    • <li>: Each item inside the list is wrapped in a list item tag (<li>)
    • </ul>: Closes the unordered list.
      Result:
        • Milk
        • Bread
        • Eggs

  • Ordered Lists (<ol>)

    • An ordered list is used when the sequence does matter-like instructions, rankings, or steps in a process. These items are number automatically.

    Basic Structure:

      <ol>
        <li>Wake Up<li>
        <li>Brush your teeth<li>
        <li>Have breakfast<li>
      <ol>

    • <ol>: Starts the ordered list (stands for ordered list)
    • <li>: Each item inside the list is wrapped in a list item tag (<li>)
    • </ol>: Closes the ordered list.
      Result:
      1. Wake up
      2. Brush your teeth
      3. Have breakfast

    • HTML Attributes and the Anchor (<a>)

    The anchor element allows you to create links that users to another webpage, a file, or even a specific section within the same page.

    Basic Structure:

    <a>Click Here</a>

    • <a> = Opening Tag
    • </a> = Closing Tag
    • Click Here = Visible Link Yet

    What Are Attribuites?

    Attributes give extra information about an HTML element. They are always added to the opening tag.

    They follow this format:
        <element attribute="value">Content</element>
    You can add multiple attributes, seperated by spaces.

    Making a Working Link with href

    The anchor tag uses attribute called href (short for Hypertext REFerence), which defines where the link should go.

    Example: Active Hyperlink

       <a href="https.//www.google.com">Visit Google </a>
    •   href="https://www.google.com/" tells the browser to go to Google when the link is clicked.
    •   Now the text "Visit Google" will appear as a blue, underlined Hyperlink.
    •   Without href, the text is just plain, unclickable text.

    Adding Multiple Attributes

    You can add more attributes to the anchor tag. For example:
       <a href="https://www.google.com" target="_blank">Open Google in a New Tab </a>

    •   taget="_blank" makes the link open in a new tab instead of the current one.
    •   You can combine as many attributes as needed, seperated by spaces.
    •   _self - Default. Opens the document in the same window/tab as it was clicked.
    •   _blank - Opens the document in a new window or tab
    •   _parent - Opens the document in the parent frame
    •   _top - Opens the document in the full body of the window

    Global Attributes

    Besides specific attributes like href, there are global attributes that work with almost any HTML tag.

    Example: draggable Attribute

    <a href="https://www.google.com/" draggable="true">Drag Me!</a>

    •   draggable="true" allows you to click and drag the link around.

    •   If you remove this attribute, the link won't be draggable anymore.
           Other global attributes include:
                •   id (for identifying an element)
                •   class (for styling with CSS)
                •   style (for inline CSS)

    HTML entities

    Some characters are reserved in HTML. If you use the less than (<) or greater that (>) signs in your HTML text, the browser might mix them with tags. Entity names or entity number scan be used to display reserved HTML characters.

    Understanding the HTML Boilerplate

    Think of this like writing a formal letter. Just as a letter has a proper format - like a header, greeting, body, and closing - an HTML document also follows a structured format. This structure ensures that web browsers understand and display your content correctly.

    What is an HTML Boilerplate?

    The HTML Boilerplate is the basic template you start with when creating any new HTML file. It includes the essential code required to set up a web page properly.

    <!DOCTYPE html>
    <html lang="en">
    <head>
         <meta charset="UTF-8">
         <meta name="viewport" content="width=device-width, initial-scale=1.0">
         <title>Document</title>
    </head>
    <body>

    </body>
    </html>

    Line-by-Line Breakdown

    1. <!DOCTYPE html>
      • This is the Document Type Declaration.
      • Is tells the browser, "Hey, this is an HTML5 document!"
    2. <html lang="en">
      • The <html> tag is the root element.
      • Everything on the web page goes inside this tag.
      • The attribute lang="en" specifies that the page is in English, which helps with accessibility and seach engines.
    3. <head> Section
      • The <head> tag contains meta-information about the webpage.
      • This is information for browsers and search engines - not the user.
        • <meta charset="UTF-8"> - Sets the character encoding to UTF-8 , allowing the page to display a wide range of characters (like emojis).
        • <meta name="viewport" content="width=device-width, initial-scale=1.0">Ensures the website looks good on mobile devices by controlling the layout.
        • <title>My First Web Page</title> Defines the title of the webpage, which appears on the browser tab.
    4. <body> Section
      • This is where the visible content of your webpage goes.
      • Whatever you write inside the <body> tags will appear on the website.
        • <h1> A heading tag, used for that main title.
        • <p> A paragraph tag for regular text.
    5. Closing Tags: </head>, </body>, and </html>
      • Always remember to close your tags to ensure your code works correclty.

    What Are Forms in HTML?

    You've filled out tons of forms in your life, At the doctor's office, at school, for activities and programs.
      If you want to sign up for something, chances are, you'll need to fill out a form.

    HTML Forms: How they Work

    In today's digital world, forms are usually build with HTML.

    • An HTML form collects user input.
    • That input is then sent to a server for processing.
    • The <form> element is used to create a form.

    Think of <form> as a container that holds different types of input fields.

    The Input Element

    One of the most important elements inside a form is <input>. It changes its behavior depending on the type attribute.

    Here are some common input types:

    •    Text Fields - <input type="text"> - Single-line text entry "My name is Bill"
    •    Radio Buttons - <input type="radio"> - Select one option "This one, the burger!"
    •    Checkboxes - <input type="checkbox"> - Select multiple options "Check! Check! Check! Check! (I'll take all the toppings!)"

    Text Field:

    • <input tpe="text"> creates a single-line text input.
    • The placeholder="My name is Bill" shows example text inside the box.

    Radio Buttons:

    • <input type="radio"> allows selecting only one option.
    • The name="toppings[]" groups them together.

    More HTML Form Elements

    The <form> element is used to collect user input and send it to a server. Inside a <form>, we can use different form elements to accept different types of user input.

    In addition to <input>, here are some other form elements you should know:

    <input> (Input Field)

    • Used to collect user input in different formats (text, email, password, number, etc).
    • Example:
        <input type="text" placeholder="Enter your name">

    Different type attributes define different behaviors, such as:

    • text - Single-line text field
    • email - Email input
    • password - Password input
    • checkbox - checkbox selection
    • radio - Radio button selection

    <label> (Label for Input Elements)

    • Describes what an input field is for. Improves accessibility.
    • Example:
        • <label for="name">Enter your Name: </label>
        <input type="text" id="name">
    • The for attribute links the <label> to an <input> field via the input's id.

    <select> (Dropdown List)

    • Creates drop-down menu where users can choose one option.
    • Example:
        • <label for="country">Choose a country: </label>
        <select id="counter" name="country">
          <option value="usa">USA</option>
          <option value="pilipins">PH</option>
          <option value="uk">UK</option>
        </select>
    • The <option> elements define each item in the dropdown.

    <button> (Clickable Button)

    • Creates a clickable button for submitting forms or performing actions.
    • Example:
        • <button type="submit">Submit</button>

    The type attribute defines behavior:

    • submit - Submits the form
    • reser - Clears the form fields
    • button - General-purpose button

    What Are Tables in HTML?

    In this lesson, you're going to learn how to create tables in HTML.

    HTML: How to Build Tables

    Let's break down the essential tools you need to create tables in HTML.

    • First, you'll use the <table> element to define a table.
    • To create rows, use <tr> elements each row gets its own <tr>.
    • Inside each row, you'll need <td> elements, which define individual table cells (columns).
    • Want headings? Use <th> instead of <td> to make column headers bold and centered by defualt.

    HTML Table Borders

    By default, tables don't have borders. If you want borders, you have to specify them using CSS.

    • The border property adds a border to your table.
    • But watch out! This can create double borders.
    • To fix tha, use border-collapse: collapse; to merge them into a single border.

    HTML: Styling Tables

    Want to make you table look even better? Here are some quick styling tips:

    • Use padding to add space inside each cell.
    • Use text-align to control the alignment of text inside cells.

    HTML: Cell and Column Spanning

    Now, what if you need a cell to stretch accross multiple columns?

    • Use the colspan attribute inside a <td> to make a cell span multiple columns.
    • Similarly, use the rowspan attribute to make a cell span multiple rows.
